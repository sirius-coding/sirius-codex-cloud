# 中国市场接单项目方案（10个生产级可交付项目）

> 说明：本方案基于统一技术栈（Spring Boot + Spring Security + Vue.js + Element Plus + MySQL + Redis + 支付宝/微信支付），给出 10 个可落地项目模板。每个项目均包含用户、商品、订单、支付、购物车、安全与合规模块，并满足高并发与生产交付标准。

---

## 0. 统一交付基线（所有项目通用）

### 0.1 架构基线
- **后端**：Spring Boot 3.x、Spring Security、Spring Data JPA/MyBatis-Plus、Spring Validation、OpenAPI 3。
- **前端**：Vue 3 + TypeScript + Vite + Element Plus + Pinia + Vue Router。
- **数据层**：MySQL 8（主从 + 读写分离）、Redis（缓存 + 分布式锁 + 会话）。
- **消息队列**：RabbitMQ / Kafka（订单状态变更、库存扣减、消息通知）。
- **对象存储**：阿里云 OSS / 腾讯云 COS（商品图、课程附件、企业文件）。
- **可观测性**：Prometheus + Grafana + Loki/ELK + SkyWalking。
- **部署**：Docker + Kubernetes（可选）+ Nginx Ingress。

### 0.2 核心功能统一模块
1. **用户管理**：注册/登录、RBAC 权限、手机号验证码、实名认证（可选）、个人资料、审计日志。
2. **商品管理**：商品/SPU/SKU、分类、标签、检索、上下架、库存、价格体系。
3. **订单管理**：下单、拆单、支付、配送/履约、退款、状态机跟踪。
4. **支付接口**：支付宝当面付/网页支付 + 微信 JSAPI/Native 支付；回调验签、防重放、防重复通知。
5. **购物车**：游客/登录态购物车、批量结算、优惠券叠加规则、价格快照。
6. **数据安全**：敏感字段加密、传输 TLS、访问控制、最小权限、脱敏日志。

### 0.3 性能目标与保障
- **并发目标**：支持 >= 10,000 并发访问。
- **响应目标**：核心接口 P95 < 3 秒。
- **策略**：
  - 热点数据 Redis 缓存 + 本地 Caffeine 二级缓存。
  - 复杂查询分页化、覆盖索引、慢 SQL 治理。
  - 订单链路异步化（MQ）、削峰填谷。
  - 限流（Sentinel/Resilience4j）、熔断降级、幂等令牌。

### 0.4 安全与中国合规要点
- 防 SQL 注入：预编译 SQL + ORM 参数化。
- 防 XSS：输入过滤 + 输出转义 + CSP。
- 防 CSRF：Token 机制（管理后台）。
- 密码：BCrypt/Argon2 存储。
- 数据加密：
  - 传输：HTTPS/TLS1.2+
  - 存储：手机号、证件号等使用 AES-GCM 字段级加密。
- 合规建议：
  - 中国大陆用户数据尽量 **境内存储**。
  - 建立隐私政策、用户授权与注销流程。
  - 按业务场景落实等保、日志留存与审计。

### 0.5 交付清单（每项目）
- 源代码（前后端 + 数据库脚本 + Dockerfile + k8s yaml）
- API 文档（OpenAPI）
- 部署文档（单机/容器/k8s）
- 测试报告（单元 + 集成 + 压测）
- 运维手册（监控告警、故障预案、备份恢复）

## 12. 测试与验收基线（每项目必须）

- **单元测试**：覆盖核心服务层、支付回调验签、订单状态机。
- **集成测试**：下单-支付-履约全链路。
- **性能测试**：JMeter/Gatling 压测 10k 并发，输出 P95/P99。
- **安全测试**：SQL 注入、XSS、越权、弱口令、接口重放。
- **验收标准**：
  - 核心接口 P95 < 3s。
  - 支付与订单链路幂等性通过。
  - 高可用演练（单点故障可恢复）。

## 13. 中国本地化 UI/业务细节清单

- 默认货币：人民币（¥），金额格式符合国内容易读写习惯。
- 日期时间：`yyyy-MM-dd HH:mm:ss`。
- 地址体系：省/市/区三级联动。
- 手机登录：+86 格式校验、短信验证码倒计时。
- 发票能力：电子发票抬头、税号、开票记录。
- 客服体系：在线客服、工单、常见问题中心。

## 14. 给甲方的实际交付建议

如果你需要“可直接生产上线”的交付方式，建议按以下节奏实施：
1. 先从以上 10 个中选 1 个主项目作为第一期（6~10 周）。
2. 复用统一技术底座与模块，后续项目可降本 30%~50%。
3. 每期都交付可运行环境：测试、预发布、生产三套配置。
4. 所有上线版本必须附带测试报告与回滚方案。
